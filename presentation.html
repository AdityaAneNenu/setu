Introduction

  • Project Overview

    SETU PM-AJAY is an end-to-end grievance and rural infrastructure tracking platform. It unifies a Django web dashboard, REST APIs, and an Expo React Native app for field operations.

  • Problem Statement

    Current village complaint workflows are mostly paper-based and difficult to monitor. Delayed updates, weak transparency, low-connectivity challenges, and language barriers reduce closure quality and trust.

Introduction

  • Motivations
    • Digitize grievance handling for underserved rural communities.
    • Enable complaint capture and synchronization in offline/low-network conditions.
    • Support multilingual and voice-assisted submission for inclusivity.
    • Bring accountability through role-based workflow and audit logs.
    • Reduce manual effort with AI-assisted photo/audio understanding.

Background & Related Work / Literature Review

  • Existing Solutions / Literature Survey
    • Paper register and manual complaint collection are still common in villages.
    • Most public complaint portals are internet-first and urban-centric.
    • Several e-governance studies highlight traceability, localization, and transparency as key adoption factors.
    • Existing mobile workflows often lack reliable offline queueing and closure verification support.

Background & Related Work / Literature Review

  • Gaps Identified / Improvements over Existing Solutions
    • Offline-first queue with retry/backoff for unstable network regions.
    • AI-assisted image and voice analysis to pre-fill complaint details.
    • Role-driven review and escalation flow across worker/manager/admin levels.
    • Evidence-based closure support with geotagged media and workflow logs.
    • Unified mobile + web visibility for faster decision-making and monitoring.

Project Objectives

  1. Create a reliable complaint lifecycle from submission to verified closure.
  2. Provide multilingual, offline-capable mobile reporting for field teams.
  3. Enable analytics and monitoring through a role-based dashboard.
  4. Use AI to improve complaint classification and data quality.

Proposed Solution

  • System Architecture / Model Diagram / Workflow / UML diagrams

Mobile App Layer

Field complaint capture, media upload, offline queueing, sync, and status updates.

→

API & Service Layer

Django + DRF APIs, Firebase-backed auth, workflow enforcement, and AI media endpoints.

→

Dashboard Layer

Monitoring, analytics, review, assignment, and administrative controls.

Workflow: Capture issue → AI-assisted extraction/classification → sync to backend → manager/admin review → field resolution with evidence → final verification and closure.

Summary

  • Key Findings

    SETU PM-AJAY demonstrates an effective digital bridge between rural complainants and government workflows with offline resilience, role-based governance, and transparent tracking.

  • Limitations

    The current implementation is prototype-scale; broader district adoption requires infrastructure hardening, larger deployment data, and deeper integrations.

  • Future Directions

    Expand multilingual UX, strengthen analytics, improve automated verification confidence, and extend integration with wider public service platforms.
